# 覆盖报告后端对接需求

- 编写日期：2026-05-26
- 面向对象：后端同学
- 目标：支持前端把覆盖报告升级为比赛展示用的“教学资源质量评审报告”
- 约束：不新增数据库表；不修改 API path；不删除现有字段；在现有 `report_json` 中追加可解释字段

## 1. 背景

当前后端覆盖报告由 `CoverageService.build_coverage_payload(generation_batch_id)` 生成，现有 `report_json` 已包含：

- `covered_knowledge_point_ids`
- `uncovered_knowledge_point_ids`
- `artifact_coverage`
- `assessment_quality`
- `important_knowledge_point_coverage`
- `warnings`

这些字段能支撑基础统计，但还不适合比赛展示。前端需要展示给评委的是一份可读的教学质量评审报告，而不是 ID 和计数。

目前前端最缺的是：

- 未覆盖知识点的名称、章节、教材证据，而不是内部 ID。
- 每类成果物的覆盖充分性，而不是只判断有没有非法引用。
- 一个知识点是否完成“课程方案 -> 教案 -> PPT -> 作业/测评”的闭环。
- 测练难度的教学语义表达，而不是裸露 `1-5` 难度值。
- 难度越界告警的具体题目、题号、题干摘要。
- 学情适配摘要，用来说明测练结构是否符合学生画像。
- 可执行补救建议。

## 2. 当前后端已有基础

根据现有代码，覆盖报告生成链路已经具备以下数据来源。

### 2.1 覆盖报告入口

- `backend/app/modules/coverage/service.py`
- 核心入口：`CoverageService.build_coverage_payload`
- 当前覆盖输入由 `_collect_artifact_references` 收集。

### 2.2 当前已统计的成果物类型

当前 `ARTIFACT_BUCKETS` 已包含：

- `curriculum_plan`：课程大纲
- `lesson_plan`：教案
- `question_item`：试卷题目
- `homework_question`：作业题目
- `courseware_slide`：课件页面

### 2.3 当前可用的知识点字段

来自 `KnowledgePoint`：

- `id`
- `point_code`
- `point_name`
- `point_type`
- `chapter_node_id`
- `importance_level`
- `difficulty_level`
- `mastery_level_hint`
- `tags_json`
- `summary_text`

### 2.4 当前可用的章节字段

来自 `ChapterNode`：

- `id`
- `title`
- `page_start`
- `page_end`

### 2.5 当前可用的教材证据字段

来自 `KnowledgeEvidence`：

- `knowledge_point_id`
- `evidence_type`
- `page_no`
- `excerpt_text`
- `score_value`

### 2.6 当前已有的难度语义

后端已有题目考查定位映射，建议直接复用，不要重新命名：

- 位置：`backend/app/shared/question_basis.py`
- 函数：`build_assessment_position`

映射关系：

- `1-2` -> `基础掌握题`
- `3` -> `典型应用题`
- `4-5` -> `综合提升题`

前端比赛展示会优先使用这组三档语义，不主展示“难度 1 / 2 / 3 / 4 / 5”。

### 2.7 当前可用的学情字段

来自 `GenerationBatch`：

- `learner_profile_version_id`

来自 `LearnerProfileRecord`：

- `subject_code`
- `score_value`
- `weakness_tags_json`
- `ability_tags_json`
- `time_plan_json`
- `summary_text`

## 3. 必须新增的 report_json 字段

以下字段全部追加到现有 `report_json` 内，旧字段继续保留。

### 3.1 `knowledge_point_summaries`

用于把覆盖范围内的知识点 ID 转换为可读摘要。覆盖范围应为当前 `knowledge_scope` 过滤后的全部知识点。

建议结构：

```json
{
  "knowledge_point_summaries": {
    "126": {
      "id": 126,
      "point_name": "闰年产生的原因",
      "point_code": null,
      "point_type": "knowledge",
      "chapter_node_id": 12,
      "chapter_title": "一 年、月、日",
      "chapter_page_range": "7-13",
      "importance_level": 3,
      "is_important": false,
      "difficulty_level": 3,
      "difficulty_band": "典型应用题",
      "mastery_level_hint": "理解",
      "tags_json": {},
      "summary_text": "...",
      "evidence": {
        "page_no": 10,
        "excerpt_text": "为了方便，把一年定为365天，叫做平年。",
        "evidence_type": "text",
        "score_value": 0.91
      }
    }
  }
}
```

实现建议：

- `difficulty_band` 复用 `build_assessment_position(difficulty_level)`。
- `is_important` 使用当前覆盖报告逻辑：`importance_level >= 4`。
- 教材证据优先取每个知识点最高 `score_value` 的 `KnowledgeEvidence`；无分数时按最小 `id` 兜底。
- 建议在 `CoverageRepository` 增加 `list_knowledge_evidence_by_point_ids(point_ids)`。

### 3.2 `uncovered_knowledge_points`

保留现有 `uncovered_knowledge_point_ids`，同时新增可读明细。

建议结构：

```json
{
  "uncovered_knowledge_points": [
    {
      "id": 126,
      "point_name": "闰年产生的原因",
      "chapter_title": "一 年、月、日",
      "difficulty_level": 3,
      "difficulty_band": "典型应用题",
      "mastery_level_hint": "理解",
      "evidence": {
        "page_no": 10,
        "excerpt_text": "为了方便，把一年定为365天，叫做平年。"
      },
      "missing_from": ["courseware_slide", "question_item", "homework_question"],
      "suggested_action": "建议补充到 PPT 讲解页，并在期末综合测中增加 1 道典型应用题。"
    }
  ]
}
```

实现建议：

- `missing_from` 基于每类成果物的覆盖情况生成。
- 不要让前端只拿到 ID 再二次查询知识点。
- `suggested_action` 第一版用规则生成，不需要 LLM。

### 3.3 `knowledge_point_coverage_matrix`

用于展示每个知识点是否完成“课程方案 -> 教案 -> PPT -> 作业/测评”闭环。

建议结构：

```json
{
  "knowledge_point_coverage_matrix": [
    {
      "knowledge_point_id": 126,
      "point_name": "闰年产生的原因",
      "chapter_title": "一 年、月、日",
      "difficulty_band": "典型应用题",
      "covered_by": {
        "curriculum_plan": { "covered": false, "reference_count": 0 },
        "lesson_plan": { "covered": false, "reference_count": 0 },
        "courseware_slide": { "covered": false, "reference_count": 0 },
        "question_item": { "covered": false, "reference_count": 0 },
        "homework_question": { "covered": false, "reference_count": 0 }
      },
      "closure_status": "no_coverage",
      "gap_types": ["not_planned", "not_taught", "not_practiced", "not_assessed"]
    }
  ]
}
```

`closure_status` 建议枚举：

- `complete_loop`：教案覆盖、PPT 覆盖，且作业或测评至少一种覆盖。
- `planning_only`：只在课程方案或教案出现，没有进入 PPT / 作业 / 测评。
- `teaching_no_assessment`：进入教案或 PPT，但没有进入作业 / 测评。
- `assessment_no_teaching`：进入作业 / 测评，但缺少教案或 PPT 支撑。
- `no_coverage`：五类成果物均未覆盖。

`gap_types` 建议枚举：

- `not_planned`
- `not_taught`
- `not_in_courseware`
- `not_practiced`
- `not_assessed`

### 3.4 `artifact_gap_analysis`

当前 `artifact_coverage` 能说明“引用数量”和“非法引用”，但不能说明覆盖是否充分。前端需要这个字段来避免把“引用有效”误展示为“覆盖正常”。

建议结构：

```json
{
  "artifact_gap_analysis": {
    "courseware_slide": {
      "display_name": "课件页面",
      "covered_count": 7,
      "total_count": 165,
      "coverage_rate": 4.24,
      "valid_reference_status": "valid",
      "coverage_status": "weak",
      "gap_summary": "课件页面仅覆盖少量知识点，当前更适合作为局部课件成果，仍需补齐更多课次 PPT。",
      "suggestions": ["继续为未生成课件的课次补齐 PPT 页面。"]
    }
  }
}
```

`valid_reference_status` 建议枚举：

- `valid`：无非法知识点引用。
- `has_invalid_refs`：存在 `invalid_knowledge_point_ids`。

`coverage_status` 建议阈值：

- `missing`：覆盖 0 个知识点。
- `weak`：覆盖率大于 0 且小于 30%。
- `partial`：覆盖率 30% 到 80%。
- `strong`：覆盖率不低于 80%。

### 3.5 `assessment_quality_v2`

保留现有 `assessment_quality`，新增教学语义版测练质量。

建议结构：

      "基础掌握题": { "count": 30, "percent": 50.0, "levels": [1, 2] },

      "综合提升题": { "count": 11, "percent": 18.33, "levels": [4, 5] }

        "scene_type": "final_exam",

### 3.7 `learner_profile_alignment`

用于支持前端展示“测练和资源是否贴合学情”。

建议结构：

```json
{
  "learner_profile_alignment": {
    "status": "available",
    "profile_version_id": 1,
    "subject_code": "math",
    "score_value": 82.0,
    "weakness_tags": ["计算能力待提升", "综合应用待提升"],
    "ability_tags": ["计算能力", "逻辑能力"],
    "time_plan_items": [
      "计划每周安排2次数学课（每次2课时），重点训练乘法运算能力和应用题分析能力"
    ],
    "summary": "当前学生数学基础中等偏上，基础计算尚可，应用题分析与逆向思维需要加强。",
    "assessment_fit_summary": "测练中应保留基础掌握题巩固计算，同时提高典型应用题占比，用于训练应用题分析能力。"
  }
}
```

实现建议：

- 根据 `GenerationBatch.learner_profile_version_id` 查询 `LearnerProfileRecord`。
- 优先取与 `Project.subject_code` 匹配的记录。
- 如果没有匹配记录，取该学情版本第一条记录。
- 学情缺失时返回：

```json
{
  "learner_profile_alignment": {
    "status": "not_available"
  }
}
```

建议在 `CoverageRepository` 增加：

- `get_project(project_id)`
- `list_learner_profile_records(profile_version_id)`

### 3.8 `action_suggestions`

用于前端展示“下一步怎么补”。

建议结构：

```json
{
  "action_suggestions": [
    {
      "priority": "high",
      "type": "add_assessment_question",
      "target_artifact_type": "question_item",
      "knowledge_point_id": 126,
      "point_name": "闰年产生的原因",
      "chapter_title": "一 年、月、日",
      "suggested_question_band": "典型应用题",
      "suggested_scene_type": "final_exam",
      "reason": "该知识点未被任何成果物覆盖，且属于理解类知识点，建议补充到期末综合测形成考查闭环。"
    }
  ]
}
```

第一版规则建议：

- 未覆盖知识点：建议先补教案 / 课件，再补测评或作业。
- 已进入教案但未进入 PPT：建议补 `courseware_slide`。
- 已进入教学资源但未进入作业 / 测评：建议补 `homework_question` 或 `question_item`。
- `difficulty_band = 基础掌握题`：优先建议课后作业。
- `difficulty_band = 典型应用题`：优先建议期末综合测或单元测。
- `difficulty_band = 综合提升题`：优先建议期末综合测。
- `importance_level >= 4`：建议优先级为 `high`。
- 与学情薄弱点关键词匹配：建议优先级至少为 `medium`。

## 4. 接口兼容要求

- 不新增数据库表。
- 不修改现有 API path。
- 不删除现有字段。
- `CoverageReportDetailResponse.report_json` 已是 `dict[str, Any]`，响应模型无需大改。
- 新字段全部追加到 `report_json`，旧前端可以继续工作。

## 5. 前端依赖优先级

P0：

- `uncovered_knowledge_points`
- `artifact_gap_analysis`
- `assessment_quality_v2`
- 增强版 `warnings`

P1：

- `knowledge_point_summaries`
- `knowledge_point_coverage_matrix`
- `learner_profile_alignment`
- `action_suggestions`

P2：

- 更精细的规则建议。
- LLM 生成质量评语。
- 按章节聚合覆盖报告。

## 6. 验收标准

后端完成后，前端应能直接展示：

- 未覆盖知识点名称，而不是 ID。
- 每个未覆盖知识点所属章节、教材页码和证据摘录。
- 每类成果物是“强覆盖 / 部分覆盖 / 弱覆盖 / 缺失”，而不是只显示“覆盖正常”。
- 测练难度用“基础掌握题 / 典型应用题 / 综合提升题”展示。
- 难度越界 warning 能看到具体题号、题型、题干摘要和场景。
- 覆盖报告能说明测练是否贴合学生学情。
- 报告能给出可执行补救建议。

## 7. 建议测试点

- 覆盖报告包含 `knowledge_point_summaries`，且未覆盖知识点能返回名称、章节、难度语义和教材证据。
- `uncovered_knowledge_points` 与 `uncovered_knowledge_point_ids` 数量一致。
- `knowledge_point_coverage_matrix` 能正确判断 `no_coverage`、`planning_only`、`teaching_no_assessment`、`assessment_no_teaching`、`complete_loop`。
- PPT 覆盖率很低但无非法引用时，`artifact_gap_analysis.courseware_slide.valid_reference_status = valid`，同时 `coverage_status = weak`。
- `assessment_quality_v2` 能把 1-5 难度聚合为三档教学语义。
- 难度越界 warning 返回题号、题型、题干摘要、当前难度语义和预期范围。
- 学情存在时返回 `learner_profile_alignment.status = available`。
- 学情记录缺失时返回 `not_available`，且报告生成成功。
- 现有覆盖报告、课后作业、测评、课件相关测试继续通过。
