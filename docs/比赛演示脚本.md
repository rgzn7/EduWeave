# EduWeave 比赛演示脚本

## 演示目标

让评委在一条完整链路里看到：教材解析有证据、教学资源能生成、多场景测练已沉淀、覆盖报告能说明质量，最后产物可以下载交付。

## 推荐演示顺序

### 1. 项目工作台

- 打开项目 `#1`，说明项目包含教材版本、学情版本、知识版本和生成批次。
- 展示“教材上传与解析”区域，选择已有教材版本和解析版本。
- 讲解解析状态、复核状态、解析策略、页数等基础信息。
- 展示“解析证据摘要”：
  - 如果后端 `evidence-summary` 已上线，重点展示 block 类型统计、图片/表格/公式统计、MinerU 参数和示例证据。
  - 如果接口尚未上线，明确说明页面已预留接口位，当前只展示真实解析版本基础信息，不展示模拟统计。

### 2. 知识抽取与批次

- 展示已就绪的知识版本，说明知识点来自确认后的解析版本。
- 展示已就绪的学情版本，说明生成过程会结合教材与学情。
- 进入最新生成批次，建议使用项目 `#1` / 批次 `#1` 或演示环境中最新 ready 批次。

### 3. 课程方案

- 打开“课程方案”页。
- 展示课程数、课时分钟、学科年级、课程概览、阶段目标、课次安排和知识点引用。
- 点击课程方案 DOCX 下载入口，说明课程方案可作为可交付文档导出。

### 4. 教案

- 打开“教案”页，切换左侧不同课次。
- 展示教学流程、教师动作、学生活动、物料清单、课后任务和知识点引用。
- 点击教案 DOCX 下载入口，说明单课教案也可导出交付。

### 5. 多场景测练

- 打开“测练”页。
- 切换“课后作业”“单元测评”“期末综合测”，说明当前不是单一 `unit_test`，而是多场景测练。
- 展示测评蓝图中的题型分布、难度分布、知识点考查权重。
- 展示试卷结果和题目列表，说明题目包含题干、答案、解析、难度和知识点来源。
- 展示“题目沉淀”区域：
  - 当前基于选中试卷真实题目做轻量筛选。
  - 后端 `question-items` 查询接口上线后可扩展为全批次题库视图。
- 点击试卷 DOCX 下载入口。

### 6. 课件

- 打开“课件”页。
- 展示课件页面结构、页面类型统计、讲稿/互动/练习等结构化内容。
- 点击 PPTX 下载入口，说明课件可直接用于课堂展示。

### 7. 覆盖报告

- 打开“覆盖报告”页。
- 展示覆盖率、警告数、课程方案/教案/试卷题目/课件页面四类覆盖矩阵。
- 讲解覆盖报告如何反查知识点是否被教学产物和测练共同覆盖。
- 如果报告已刷新，说明它纳入了 `question_item` 和 `courseware_slide`，不是只统计课程方案与教案。

## 收尾话术

EduWeave 的主线不是单次生成一个文件，而是把教材解析、知识抽取、教学资源、多场景测练、覆盖报告和文件导出连成闭环。解析证据摘要已经可以展示真实结构化数据；后端继续补齐题库查询后，前端题目沉淀区可以进一步升级为全批次题库视图。
