# EduWeave 演示数据检查清单

## 基础数据

- 项目 `#1` 存在，并能进入项目工作台。
- 项目中至少有一个教材版本，解析状态为 `success` 或可解释的完成态。
- 至少有一个解析版本完成确认，`review_status` 为 `confirmed`。
- 至少有一个学情版本完成抽取，状态为 `success` 或 ready。
- 至少有一个知识版本 ready，知识点数量不为 0。

## 生成批次

- 项目中至少有一个可演示批次，优先使用批次 `#1`。
- 批次绑定了课程方案 ID。
- 批次包含至少一份教案。
- 批次状态为 ready/success，或任务状态足以解释当前生成进度。

## 多场景测练

- 至少有 `unit_test` 场景试卷。
- 至少再准备一个其他场景，优先 `homework`，其次 `final_exam`。
- 每个演示场景尽量同时具备测评蓝图和试卷结果。
- 试卷结果中应包含题目明细，字段至少包括题干、答案、解析、题型、难度。

## 解析证据

- 检查后端 `evidence-summary` 接口能返回：
  - 页数、block 总数、issue 数。
  - block 类型统计，如 heading、paragraph、table、image、formula。
  - 图片、表格、公式或资源引用统计。
  - MinerU 参数摘要。
  - 至少 3 条示例证据，包含页码、block 编号、类型和文本片段。
- 如果接口异常或当前解析版本没有统计数据，前端应展示降级状态，不能显示模拟 block 统计。

## 覆盖报告

- 覆盖报告已刷新。
- 覆盖矩阵至少包含四类产物：
  - 课程方案。
  - 教案。
  - 试卷题目。
  - 课件页面。
- 覆盖报告能展示覆盖率、警告数、知识点覆盖明细。
- 若存在难度或策略校验警告，演示时准备一句解释说明。

## 文件导出

- 课程方案 DOCX 可点击下载。
- 教案 DOCX 可点击下载。
- 试卷 DOCX 可点击下载。
- 课件 PPTX 可点击下载。
- 若文件尚未导出，首次点击应触发导出并返回下载地址。

## 端测路径

- 使用 Chrome 插件或 Computer Use 打开前端页面。
- 进入项目工作台，检查教材解析、解析证据、知识版本、生成批次。
- 进入批次详情，依次切换课程方案、教案、测练、课件、覆盖报告。
- 触发课程方案、教案、试卷、课件下载入口。
- 在测练页切换场景，并操作题目沉淀筛选。
- 在后端接口异常或空数据时确认页面不崩溃、不展示假数据。
