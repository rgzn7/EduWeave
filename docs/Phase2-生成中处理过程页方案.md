# Phase 2：生成中处理过程页方案

## 定位

Phase 2 是“正在生成 / 待处理 / 失败待处理”的处理过程页，用来让老师知道 EduWeave 正在做什么、当前停在哪一步、下一步需要什么操作。

它不是历史页，也不是成果详情页。已完成备课应该进入成果页查看课程方案、教案和覆盖报告，并按需生成 PPT 课件与配套测练。

## 入口规则

- 首页负责新建备课，并在下方展示“最近备课”：最多展示 3 个真实记录，未完成优先，其次展示最近完成的备课。
- 正在生成、待确认、失败待处理的备课进入 `/projects/:projectId`。
- 只有最近批次状态为 `success` 的备课，才从首页“最近备课”或“备课记录”进入 `/projects/:projectId/batches/:batchId`。
- 最近批次为 `pending` 或 `processing` 的备课进入 `/projects/:projectId`，继续显示生成过程。
- 最近批次为 `failure` 的备课进入 `/projects/:projectId`，显示需要处理。
- “备课记录”页面展示全部状态的备课，包括正在生成、待补充、已完成和未开始。
- Phase 2 不提供“更换材料”或“重新开始备课”。如果材料选错，用户回首页重新创建一次备课。

## 页面结构

处理过程页只展示本次备课的输入材料和五步进度：

1. 上传材料：展示本次使用的教材 PDF 和学情 DOCX。
2. 理解教材：展示页数、图表、公式和解析证据同步状态。
3. 分析学情：展示学生基础、薄弱点、学习习惯和课时安排。
4. 整理教学重点：展示章节数、知识点数和重点知识。
5. 生成备课资源：展示课程方案、多课教案和覆盖报告是否准备好；PPT 按课生成，测练按场景生成。

右侧只保留一个当前行动面板。完成态只出现一个主按钮：`查看备课资源`。

## 不做内容

- 不展示成果正文，成果查看属于 Phase 3。
- 不展示历史消息流。
- 不展示任务、批次、版本、JSON、后端字段或原始错误。
- 不提供材料替换入口。
- 不做演示快进回放，回放属于后续独立 Phase。
- 首页“最近备课”不是演示区，不重复项目、不造假数据；演示回放后续单独设计。

## 验收标准

- `/projects/:projectId` 没有 `更换材料`、`重新开始备课`、`技术细节`。
- 页面标题去掉 `.pdf/.docx/.pptx` 后缀。
- 首页下方显示“最近备课”和“查看更多”，最多 3 个真实项目。
- 左侧菜单显示“备课记录”。
- `/history` 页面标题显示“备课记录”，并展示全部状态。
- 已完成项目默认进入成果页。
- 刚创建批次但批次未成功时，不显示“已完成”，不进入成果页。
- PPT 统一表达为按课生成，测练统一表达为按场景生成。
- 未完成项目进入处理过程页。
