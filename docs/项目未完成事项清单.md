## 未完成事项

### 1. 解析证据摘要真实数据接入

状态：已完成后端接口，前端已按最新字段适配

接口：

```http
GET /api/v1/parse-versions/{parse_version_id}/evidence-summary
```

用途：

- 支撑教材解析页“解析证据”区域展示 MinerU 解析能力。
- 让评委能看到解析版本、block 统计、资源统计、MinerU 参数和示例证据。

后端当前返回内容：

- 解析版本基础信息：`parse_version_id`、`strategy_code`、`mineru_model`、`parse_status`、`review_status`
- 规模统计：`volume.page_count`、`volume.block_count`、`volume.issue_count`
- block 类型统计：`block_type_counts`
- 多媒体/结构统计：`volume.image_block_count`、`volume.table_block_count`、`volume.equation_block_count`、`volume.asset_block_count`
- MinerU 参数摘要：`mineru_parameters`
- 示例证据：`sample_blocks`

前端当前状态：

- 已接入 `ParseEvidenceSummary` 最新字段。
- 已接入 `getParseEvidenceSummary(parseVersionId)` API wrapper。
- 教材解析页已增加“解析证据”区域。
- 接口失败或空数据时仍保留降级提示，不展示虚构统计。

验收标准：

- 项目工作台教材解析页可以展示真实 block 类型统计。
- 图片、表格、公式等资源统计来自后端真实数据。
- 示例证据能对应到真实页码和 block。
- 接口失败时页面不崩溃，有明确等待/失败提示。

### 2. 题库查询真实数据接入

状态：已完成后端接口，前端已接入全批次题库查询

接口：

```http
GET /api/v1/question-items
```

用途：

- 支撑“题目沉淀/题库基础视图”从当前试卷题目展示升级为全批次题库查询。
- 展示系统具备题目沉淀能力，而不是只生成一次性试卷。

已支持筛选参数：

- `generation_batch_id`
- `paper_result_id`
- `knowledge_point_id`
- `question_type`
- `difficulty_level`
- `scene_type`
- `page`
- `page_size`

返回内容：

- 题干
- 答案
- 解析
- 题型
- 难度
- 知识点
- 所属试卷
- 测练场景

前端当前状态：

- 已接入 `QuestionBankItem` 类型。
- 已接入 `listQuestionItems(params)` API wrapper。
- 测练页“题目沉淀”已使用全批次题库数据。
- 接口失败时保留当前试卷详情题目作为降级视图。

验收标准：

- 测练页可以按批次查看题目沉淀列表。
- 支持按题型、难度、知识点、测练场景筛选。
- 能区分 `unit_test`、`homework`、`final_exam` 等不同测练场景来源。
- 后端接口异常时，当前试卷内题目展示仍可正常使用。
