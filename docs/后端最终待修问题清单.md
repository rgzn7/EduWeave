# 后端最终待修问题清单

## 摘要

当前项目主链路已经基本闭环，前端构建通过，后端全量测试在本地指定测试数据库端口后通过。远端后端已补齐解析证据摘要接口，下面只保留后端仍需要补齐或修复的事项，不包含本地 MySQL 测试端口问题。

本清单目标是让前端演示从“可讲清楚”升级到“数据真实可展示”，并降低覆盖报告在异常数据下的失败风险。

## 需要后端处理的问题

### 1. 补题库查询接口

优先级：P1

建议接口：

```http
GET /api/v1/question-items
```

当前状态：

- 后端已有试卷结果列表和试卷详情接口。
- 试卷详情里能返回当前试卷题目。
- 但没有面向题库沉淀的 `question_item` 全局查询接口。
- 前端目前只能基于当前试卷详情做轻量题目展示，不能展示全批次题库视图。

建议支持筛选参数：

- `generation_batch_id`
- `paper_result_id`
- `knowledge_point_id`
- `question_type`
- `difficulty_level`
- `scene_type`
- `page`
- `page_size`

建议返回字段：

- `id`
- `generation_batch_id`
- `paper_result_id`
- `knowledge_point_id`
- `question_no`
- `question_type`
- `difficulty_level`
- `score_value`
- `stem_text`
- `options_json`
- `answer_text`
- `analysis_text`
- `source_trace_json`
- 所属试卷标题
- 所属测练场景 `scene_type`
- 创建时间和更新时间

验收标准：

- 支持按批次查询题目列表。
- 支持按题型、难度、知识点、场景筛选。
- 返回数据必须做当前教师权限校验。
- 分页结构和现有列表接口保持一致。

### 2. 修复覆盖报告课件页码解析不稳

优先级：P2

当前风险：

覆盖报告收集课件页面时，当前逻辑直接将 `slide_no` 转成整数。如果课件结构中 `slide_no` 是非数字字符串，覆盖报告刷新会抛异常。

影响：

- 手动刷新覆盖报告可能失败。
- 测评生成或课件生成后自动刷新覆盖报告时也可能失败。
- 虽然正常生成数据一般不会触发，但异常数据、人工编辑数据或第三方返回变化都可能触发。

建议修复：

- 增加安全整数解析函数。
- `slide_no` 缺失或无法解析时 fallback 到当前遍历序号 `slide_index`。
- 不要因为单页课件页码异常导致整个覆盖报告失败。

验收标准：

- `slide_no` 为整数时保持原值。
- `slide_no` 为字符串数字时可正常解析。
- `slide_no` 为非数字字符串、空值或异常类型时 fallback 到 `slide_index`。
- 覆盖报告刷新接口仍返回 200。

### 3. 修正覆盖报告课件统计口径

优先级：P2

当前风险：

覆盖报告当前可能统计批次下全部课件结果。如果同一批次存在失败课件、旧课件或重复生成课件，覆盖矩阵可能混入不该展示的课件页面。

影响：

- 覆盖报告中的课件页面数量可能不准确。
- 评委看到的覆盖矩阵可能和当前有效课件不一致。
- 演示环境数据干净时问题不明显，但长期使用或多次重试后容易出现偏差。

建议修复：

- 只纳入 `result_status=success` 的课件结果。
- 如果同一批次同一教案存在多份成功课件，建议只取最新成功课件。
- 或者后端明确增加“当前有效课件”状态，并只统计有效版本。

验收标准：

- 失败课件不会进入覆盖矩阵。
- processing / pending 课件不会进入覆盖矩阵。
- 同一教案多次生成课件时，覆盖报告口径明确且稳定。
- 覆盖报告中的课件页面数量和前端展示的有效课件一致。

### 4. 同步后端文档中的过期描述

优先级：P3

当前问题：

后端说明文档里仍存在过期描述，例如：

- 测评默认只是 `unit_test`。
- 试卷暂不提供 DOCX 导出。

实际状态：

- 后端已支持多场景测练：
  - `homework`
  - `unit_test`
  - `final_exam`
- 试卷 DOCX 导出已支持。
- 覆盖报告已纳入题目和课件页面。

需要同步的文档：

- 后端 README 或后端说明文档。
- API 文档。
- 如有比赛交付说明，也要同步。

验收标准：

- 文档不再保留“测评仅 unit_test”或“试卷暂不导出 DOCX”的旧描述。
- 文档明确说明多场景测练能力。
- 文档明确说明课程方案、教案、试卷 DOCX 和课件 PPTX 均可下载。
- 文档明确说明 `question_item` 可作为题库基础能力。
