## 1. 文档说明

本文档用于定义 EduWeave AI项目的业务目标、系统范围、功能需求、非功能需求、数据需求、外部集成要求与阶段性交付边界，为后续的系统架构设计、数据库设计、接口设计、任务拆解、原型设计与验收测试提供统一依据。

本文档聚焦“需求规格”层，不直接替代详细设计文档、数据库表结构文档、接口文档和部署文档。涉及实现方式时，仅保留与需求边界强相关的约束说明。

## 2. 项目背景

在教育办公与教学服务场景中，教材 PDF、学情分析文档、教学参考资料等内容大多以非结构化文件形式存在。教师或教研人员往往需要花费大量时间，手动完成教材阅读、知识提炼、课程规划、教案编写、课件制作、课后作业设计和试卷编制等工作，存在以下共性问题：

- 教材版式复杂，包含跨栏排版、图表、公式、流程图与跨页结构，传统文本抽取难以稳定复原。
- 教材解析后的内容往往是碎片化文本，难以直接转换成符合教学逻辑的课程大纲和教案。
- 学情信息分散在 Word 文档中，难以系统性融入教学规划与测练生成。
- 从教材到 PPT、作业、试卷的生产链路依赖人工，效率低且一致性难以保障。
- 已生成的教学资源缺乏知识点覆盖校验与来源追溯，难以满足教学质量控制要求。

因此，本项目需要建设一套面向教育场景的全链路智能资源重构平台，将教材与学情文件转化为结构化教学资产，并自动生成可直接交付的教学成果物。

## 3. 建设目标

### 3.1 总体目标

构建一套以教材文档与学情分析文件为输入，以课程大纲、教案、课堂课件、课后作业、测试试卷与覆盖率分析报告为输出的 AI 教学资源生产系统，实现“解析、理解、规划、生成、校验、交付”的全流程闭环。

### 3.2 业务目标

- 将教材 PDF 和学情分析文件转化为可检索、可追溯、可复用的结构化教学数据资产。
- 基于教学目标摘要、学生画像与时间规划，自动生成多课时课程规划与教案。
- 自动生成符合教学逻辑的课堂 PPT、课后作业和测试试卷。
- 对生成结果进行知识点覆盖率与难度分布校验，提升资源质量可控性。
- 建立标准化生产链路，使不同教材版本、不同学科、不同学生画像能够复用同一系统流程。

### 3.3 产品目标

- 支持教学资源文件统一上传、存储、解析、管理与版本追踪。
- 支持任务化、异步化的资源生成流程。
- 支持结构化结果查看、人工调整与再次生成。
- 支持输出成果物的在线预览、下载与归档。

## 4. 系统定位与范围

### 4.1 系统定位

本系统定位为教育内容生产平台，不是单一的 PDF 解析工具，也不是单纯的大模型问答工具。系统核心价值在于：

- 以 MinerU 为文档解析入口，完成高保真内容提取。
- 以 MySQL 为业务结构化数据中心，承载教材、学情、知识点、规划结果与任务状态。
- 以华为云 OBS 为文件资产中心，承载原始文档、中间产物与最终交付物。
- 以 Milvus 为语义检索中心，承载教材分块、知识点与范例内容的向量化检索能力。
- 以大模型能力为教学规划与资源生成引擎，形成可解释、可追溯的教学资源输出。

### 4.2 本期建设范围

本期需求覆盖以下能力：

- 教材文档上传、存储、解析与结构化管理。
- 学情分析文档上传、结构化抽取与画像管理。
- 教材知识结构抽取与知识点矩阵构建。
- 基于教材与学情的课程大纲与教案生成。
- 基于规划结果的课件、作业、试卷生成。
- 覆盖率分析与结果追溯。
- 文件资产管理、任务状态跟踪、成果归档与下载。

### 4.3 暂不纳入本期范围

以下内容原则上不作为本期必做需求，可作为后续扩展方向：

- 多教师组织协作、权限审批流与多租户企业化能力。
- 学生端在线做题、自动判分与学习闭环。
- 实时课堂互动、直播授课、语音讲解与数字人讲课。
- 复杂 BI 分析大屏与经营报表。
- 教务排课、排班、收费、CRM 等非本赛题核心能力。

## 5. 用户角色

本系统本期采用单一教师角色：

- 教师：上传教材与学情，发起课程规划和资源生成任务，查看并下载产物；同时负责维护知识结构、审核规划结果、调整模板与题目策略等教学配置工作。

为贴合赛题范围，本期不强制拆分独立的“教研人员”角色，相关职责默认归属于教师。若后续项目产品化，可再将教师细分为授课教师、备课教师或教研人员等角色。

## 6. 业务流程概述

系统的标准业务流程如下：

1. 教师登录系统。
2. 教师创建项目或教学任务。
3. 教师上传教材 PDF、学情分析文档及可选参考文件。
4. 系统将原始文件保存到 OBS，并在 MySQL 中记录文件元数据与归属关系。
5. 系统发起教材解析任务，调用 MinerU 输出结构化内容与中间产物。
6. 系统解析学情文档，抽取学生画像、教材版本、学科成绩、能力特征与时间规划。
7. 系统基于解析结果抽取章节树、知识点、教学目标摘要与重难点，形成教材知识结构。
8. 系统根据学情画像、课时规划与教学目标摘要生成课程大纲、教案和测评蓝图。
9. 系统根据规划结果生成课堂课件、课后作业、测试试卷等资源。
10. 系统对生成结果执行覆盖率分析、难度校验与来源追溯。
11. 教师查看结果，必要时调整参数并二次生成，最终下载或归档成果物。

## 7. 核心业务对象

系统至少需要围绕以下业务对象建立统一模型：

- 项目

- 教材文档

- 知识点

- 题目蓝图

- 试卷
- 课件

- 生成追溯记录

## 8. 功能需求

### 8.1 项目与任务管理

系统应支持以“项目”或“教学任务”为组织单元管理教材、学情、解析结果与生成产物。

功能要求如下：

- 支持创建项目，并维护项目名称、学科、年级、教材版本、适用对象、备注等基础信息。
- 支持将多个文件资产关联到同一个项目。
- 支持查看项目下的解析任务、生成任务、产物清单与最近更新时间。
- 支持记录任务状态，包括待处理、处理中、成功、失败、部分成功、已取消。
- 支持查看任务日志、失败原因与重试入口。

### 8.2 教材文件管理

系统应支持教材 PDF 的上传、校验、存储和版本管理。

功能要求如下：

- 支持上传 PDF 教材文件，并校验文件格式、大小和基础完整性。
- 支持自动识别教材名称、出版社、学科、年级、册别等信息；识别失败时允许人工补录。
- 支持同一教材的多版本上传与版本区分。
- 支持记录教材文件在 OBS 中的对象路径、哈希值、上传时间、上传人和版本号。
- 支持查看原始教材文件并进行下载。

### 8.3 学情文件管理

系统应支持学情分析文档的上传、解析与结构化管理。

功能要求如下：

- 支持上传 doc、docx 等学情文档。
- 支持抽取学生姓名或匿名标识、地区、年级、学习科目、教材版本、分数、能力描述、行为特点与时间规划。
- 支持将学情文档中的多科目信息拆分为按学科管理的学生学情记录。
- 支持对抽取结果进行人工修正。
- 支持记录学情文档在 OBS 中的对象路径、版本信息与解析状态。

### 8.4 文档解析

系统应支持基于 MinerU 的教材高保真解析，输出可供后续结构化和生成使用的中间结果。

功能要求如下：

- 支持发起教材解析任务，并异步执行。
- 支持解析页级文本、标题、段落、表格、图片、公式、列表、章节结构等内容。
- 支持保存 MinerU 输出的中间结果，如 markdown、json、图片切片、表格图片和公式相关资源。
- 支持记录每次解析的模型或配置参数、执行时间、任务耗时与状态。
- 支持查看解析结果预览，并能够定位原始页码。
- 支持对解析失败、缺页、乱码、表格丢失、图表缺失等异常情况进行标记。
- 支持按页级对异常内容进行局部重解析，避免整本教材重复处理，并保留重解析前后的版本差异。
- 支持对公式、跨页表格、实验流程图等复杂内容进行异常标记，并纳入人工复核流程。
- 支持提供原始页图、解析文本、结构块坐标的对照预览，便于人工复核与修正。
- 支持将人工修正后的解析结果写回为新的解析版本，并作为后续知识抽取的默认输入版本。

#### 8.4.1 解析异常处理与人工复核流程

系统应建立面向复杂教材的解析纠错闭环，至少包括以下步骤：

1. 自动识别异常：检测缺页、乱码、结构块缺失、标题层级错乱、公式截断、表格断裂、图表说明缺失等问题，并输出异常类型、页码、块坐标和严重级别。
2. 局部重解析：支持按页级重新触发解析，并保留重解析前后结果与差异记录。
3. 人工复核修正：支持教师基于原始页图、解析文本和结构块对照视图，修正标题、正文、表格、公式、图表说明等内容。
4. 修正结果生效：人工确认后的结果应写回新的解析版本，并作为后续知识抽取的默认输入；课程规划、课件生成和测练生成通过其绑定的知识版本继承该结果。

### 8.5 知识结构化

系统应支持将解析结果转换为可用于教学规划和资源生成的知识结构。

功能要求如下：

- 支持从教材解析结果中抽取单元、章节、课文、知识点、重难点和教学目标摘要。
- 支持建立知识点与教材章节、页码、原文片段之间的映射关系。
- 支持提取示例题、实验步骤、活动任务、习题与相关图表说明。
- 支持形成教材知识点矩阵，至少包含知识点名称、所属章节、难度建议、重要度和来源证据。
- 支持对知识点进行人工增删改与归并。

### 8.6 学情画像建模

系统应支持把学情文档中的信息转化为可参与规划和生成的结构化画像。

功能要求如下：

- 支持按学生、按学科维护学情画像。
- 支持提取学生优势、薄弱点、专注情况、表达能力、计算能力、阅读能力、逻辑能力和学习习惯等标签。
- 支持提取培训频次、单次课时、总课次和阶段目标等时间规划信息。
- 支持基于结构化字段与文本摘要同时保存学情结果。
- 支持学情与教材、年级、学科之间的关联校验。

### 8.7 课程大纲生成

系统应支持根据教材知识结构与学情画像自动生成课程大纲。

功能要求如下：

- 支持选择教材、学生画像、目标学科与目标章节范围发起课程大纲生成。
- 支持生成课程概述、课时安排、章节目标、能力提升目标与阶段性成果。
- 支持课程大纲按总课次、单次时长、学习重点和学生薄弱点自动调整。
- 支持输出结构化课程大纲数据，并支持导出为文档。
- 支持人工编辑课程大纲，并保留版本。

### 8.8 教案生成

系统应支持在课程大纲基础上生成教师讲义和教案。

功能要求如下：

- 支持生成课程概述，包括课型、授课对象、授课时长、授课目标和物料清单。
- 支持生成核心知识概述与知识讲解顺序。
- 支持生成标准行课流程，至少覆盖课前互动、导入、讲解、练习、总结与课后安排。
- 支持根据学生画像调整讲解深度、例题数量、互动方式和作业建议。
- 支持导出教案为 docx 或 pdf 等文件格式。

### 8.9 测评蓝图与权重配置

系统应支持在生成题目之前建立知识点测评蓝图。

功能要求如下：

- 支持根据课程目标、章节权重和学生能力情况建立测评蓝图。
- 支持为知识点设置考查权重、题量建议、题型建议和难度区间。
- 支持题型覆盖选择，如选择题、填空题、计算题、阅读理解题、应用题、实验题等。
- 支持对不同课时、单元和阶段设置不同测评策略。
- 支持人工调整测评蓝图并保存。

### 8.10 课件生成

系统应支持根据课程大纲、教案和模板生成课堂课件。

功能要求如下：

- 支持选择课件模板并生成成品 PPT。
- 支持生成课件封面、目录页、知识讲解页、例题页、互动页、总结页和作业页等页面类型。
- 支持将教材中的图文、表格、流程说明、知识点摘要和例题内容编排到课件页面中。
- 支持根据学生画像调整每页信息密度、讲练比例和表现形式。
- 支持生成结构化幻灯片数据，并导出为 pptx 或可继续编辑的中间格式。
- 支持保存生成产物到 OBS，并记录版本与下载地址。

### 8.11 作业与试卷生成

系统应支持根据测评蓝图自动生成课后作业和测试试卷。

功能要求如下：

- 支持按知识点权重、题型、难度和题量生成作业或试卷。
- 支持生成题干、答案、解析、知识点标签和难度标签。
- 支持按照课后作业、单元测试、阶段测验、期末测试等不同场景生成不同结构的试题集合。
- 支持避免题目内容与教材目标脱节，题目应能关联到知识点与教材出处。
- 支持导出为 docx、pdf 或内部题库格式。

### 8.12 覆盖率分析与质量校验

系统应支持对生成结果进行覆盖率分析与基础质量校验。

功能要求如下：

- 支持统计课程大纲、教案、课件和试卷所覆盖的知识点集合。
- 支持输出知识点覆盖率分析表，展示已覆盖、未覆盖、重复覆盖与重点覆盖情况。
- 支持分析试题的题型分布、难度分布和章节分布。
- 支持展示每个生成结果对应的知识点来源和教材证据片段。
- 支持对明显不一致情况进行告警，如题目知识点缺失、章节覆盖不足、重点章节权重偏低等。

### 8.13 语义检索与知识召回

系统应支持基于 Milvus 的向量检索能力，以提升知识召回质量和生成可解释性。

功能要求如下：

- 支持将教材分块、知识点说明、典型题目、模板片段等内容进行向量化存储。
- 支持基于查询语义召回相关教材片段与知识点证据。
- 支持将召回结果用于教案、课件和试题生成。
- 支持记录每次生成任务引用了哪些检索结果。
- 支持在向量库异常时提供告警、重试或受控降级机制。

### 8.14 文件资产管理

系统应支持对原始文件、中间文件和最终交付物进行统一管理。

功能要求如下：

- 支持原始文件、中间解析结果和最终成果物统一存储到华为云 OBS。
- 支持记录对象路径、文件类型、文件大小、上传来源，并通过业务表关联所属业务对象与版本关系。
- 支持对不同类型文件进行分类浏览和下载。
- 支持同一业务对象的多版本成果物归档。
- 支持删除策略、覆盖策略或保留策略的配置。

### 8.15 追溯与审计

系统应支持对生成内容的来源进行追溯，满足演示可解释与质量复核需求。

功能要求如下：

- 支持记录课程大纲、教案、课件页、题目等内容对应的教材来源、知识点来源和学情依据。
- 支持查看单个成果物的生成参数、模型版本、输入来源和任务链路，核心信息可通过批次快照与任务记录组合展示。
- 支持保存人工调整记录与版本变更记录。
- 支持基础操作审计，如上传、解析、生成、重试、导出等行为。

### 8.16 认证与访问控制

系统应支持面向教师的基础身份认证与访问控制能力，作为项目、文件、解析和生成等业务能力的前置支撑。

功能要求如下：

- 支持基于用户名和密码的登录认证。
- 支持返回访问令牌，访问令牌失效后允许重新登录获取新令牌。
- 支持获取当前登录用户信息，包括用户标识、显示名称和账号状态。
- 支持退出当前使用终端的登录态；本期默认采用清理本地令牌并重新登录的轻量方案，不强制要求服务端持久化登录会话。
- 支持对未登录、令牌无效、令牌过期和账号被禁用等场景返回明确错误信息。
- 本期不开放公开注册能力，用户账户默认由系统初始化预置。

## 9. 外部集成需求

### 9.1 MinerU 集成

- 系统应支持调用 MinerU 进行教材解析。
- 系统应支持处理 MinerU 的结构化输出及其中间文件。
- 系统应支持对解析失败和异常结果进行任务级处理。
- 系统应支持对异常页进行局部重解析，并记录重解析版本与差异。
- 系统应支持对公式、跨页表格、实验图表等复杂内容进行异常标记，并纳入人工复核流程。

### 9.2 华为云 OBS 集成

- 系统应支持通过统一存储服务接入华为云 OBS。
- 系统应支持上传、读取、下载和版本归档能力。
- 系统应支持对接 AK/SK 鉴权配置、Bucket 配置、Endpoint 配置和对象路径规划。

### 9.3 MySQL 集成

- 系统应使用 MySQL 作为核心业务数据库。
- 系统应支持教材、学情、知识点、任务、产物、日志等结构化数据持久化。
- 系统应支持后续进行索引优化、分页查询和统计分析。

### 9.4 Milvus 集成

- 系统应接入 Milvus 作为语义检索基础设施。
- 系统应支持集合创建、向量写入、向量删除、语义检索和元数据过滤。
- 如后续采用 Zilliz 托管环境，仍应按 Milvus 兼容协议完成接入。

### 9.5 大模型服务集成

- 系统应支持调用大模型服务完成知识抽取、规划生成、课件生成和试题生成。
- 系统应支持对不同任务使用不同提示词模板与模型配置。
- 系统应支持对生成失败、超时和结果不完整场景进行重试或回退处理。

## 10. 非功能需求

### 10.1 性能要求

- 系统应支持单个教材解析任务的异步处理，用户无需阻塞等待。
- 系统应支持分页查看教材、知识点、题目和产物列表。
- 系统应支持基础并发场景下的任务提交与结果查询。
- 系统应支持对大文件上传与下载进行稳定处理。

### 10.2 可用性要求

- 系统应具备清晰的任务状态反馈机制。
- 系统应在解析失败、生成失败、外部服务异常时给出明确错误信息。
- 系统应支持失败任务重试。

### 10.3 可维护性要求

- 系统应采用模块化设计，解析、知识、规划、生成、存储能力应保持清晰边界。
- 系统应支持后续扩展更多教材类型、更多模板和更多生成产物类型。
- 系统应支持配置化管理存储、模型与向量库连接。

### 10.4 一致性要求

- 教案、课件、作业和试卷应与课程大纲、知识点矩阵保持逻辑一致。
- 生成的知识点、章节和学情应能够互相映射，不得出现明显错配。
- 同一次生成任务的多个成果物应使用统一的目标范围与版本基线。

### 10.5 安全性要求

- 系统应支持基础身份认证与访问控制能力。
- 系统应避免敏感配置明文暴露在前端。
- 系统应对文件上传类型、大小和来源进行校验。
- 系统应保留后续接入更细粒度权限体系的能力。

### 10.6 可观测性要求

- 系统应记录关键任务日志、接口日志与异常日志。
- 系统应支持追踪单次任务的执行链路和耗时情况。
- 系统应支持查看外部依赖调用状态。

### 10.7 关键质量指标

为保证系统满足比赛演示、实现落地与后续验收要求，本期除功能可用外，还应满足以下关键质量指标。除特别说明外，以下指标均以“单本 200 页以内教材 PDF + 单份学情文档 + 生成 1 份课程大纲、1 份教案、1 套课件、1 套测练与 1 份覆盖率报告”的标准演示场景进行统计。

| 指标项 | 指标定义 | 目标值 | 统计说明 |
| --- | --- | --- | --- |
| 教材结构识别完整率 | 对标题、段落、列表、表格、图片、公式等结构块的识别完整程度 | >= 90% | 采用人工抽检与系统统计结合方式评估 |
| 章节树识别准确率 | 单元、章节、课文层级与教材原始目录及正文结构的一致程度 | >= 90% | 抽检目录页与正文页进行比对 |
| 公式还原可用率 | 公式内容可正确辨识并可供后续知识抽取使用的比例 | >= 85% | 重点抽检理科教材中的公式区域 |
| 表格还原可用率 | 表格行列结构及关键字段可被正确识别的比例 | >= 85% | 包含跨页表格场景 |
| 知识点抽取命中率 | 人工标注的核心知识点中被系统正确抽取的比例 | >= 85% | 以章节样本进行人工复核 |
| 题目知识点关联准确率 | 生成题目绑定到正确知识点与章节的比例 | >= 85% | 抽检题干、答案、解析与标签 |
| 重点知识点覆盖率 | 测评蓝图中的重点知识点被课件或测练覆盖的比例 | >= 90% | 以覆盖率分析报告统计结果为准 |
| 难度分布偏差 | 生成题目难度分布与测评蓝图目标分布的偏差 | <= 15% | 按题量占比进行计算 |
| 覆盖率报告完整性 | 报告中已覆盖、未覆盖、重复覆盖、重点覆盖、来源证据等关键字段的完整程度 | = 100% | 缺任一关键字段视为不通过 |
| 教材解析任务完成时长 | 单本教材从发起解析到输出可用结构化结果的耗时 | <= 8 分钟 | 标准演示场景下统计 |
| 端到端生成时长 | 从上传完成后到课程大纲、教案、课件、测练与覆盖率报告全部生成完成的总耗时 | <= 20 分钟 | 单任务标准演示场景下统计 |

## 13. 验收需求

系统达到以下标准时，可视为满足本期核心需求：

### 13.1 输入处理验收

- 能成功上传教材 PDF 和学情文档。
- 能将文件写入 OBS，并在 MySQL 中保存元数据。
- 能成功发起并完成教材解析任务。
- 能从学情文档中抽取基础画像信息。

### 13.2 结构化处理验收

- 能从教材中识别章节结构与核心知识点。
- 能将知识点与教材章节、页码或原文片段建立映射。
- 能生成可编辑的学生学情画像与课程目标输入。

### 13.3 生成能力验收

- 能基于教材和学情生成课程大纲。
- 能基于课程大纲生成教案。
- 能生成至少一种完整课件产物。
- 能生成至少一种作业或试卷产物。

### 13.4 校验与追溯验收

- 能输出知识点覆盖率分析结果。
- 能展示至少一种成果物与教材知识点之间的来源映射。
- 能追踪单次任务的执行状态和生成链路。

### 13.5 量化验收标准

除满足上述功能性验收项外，本期验收还应满足以下量化要求：

- 验收样本不得少于 3 组“教材 + 学情”组合，其中至少 1 组教材应包含公式、复杂图表或跨页表格等复杂内容。
- 每组样本应完整跑通上传、解析、知识结构化、课程大纲生成、教案生成、课件生成、测练生成与覆盖率分析全流程。
- 第 10.7 节中“覆盖率报告完整性”指标必须达到 100%，其余关键质量指标应达到对应目标值。
- 验收过程中应至少演示 1 次解析异常纠错闭环，流程应覆盖异常识别、页级重解析、人工修正、修正结果生效与后续生成结果更新。
- 若任一关键质量指标未达标，系统应能够输出对应任务日志、异常原因、补救策略与复验结果，方可视为阶段性通过。

### 13.6 认证与访问控制验收

- 能使用预置教师账号完成登录。
- 登录成功后，受保护接口能够基于令牌访问，未登录请求会被正确拦截。
- 能获取当前登录用户信息并确认账号状态正常。
- 令牌失效后，用户需要重新登录才能继续访问受保护接口。

## 14. 约束与假设

- 本项目默认面向 K12 教育场景，优先支持语文、数学、英语、科学等教材。
- 本项目默认教材与学情文件由教师上传，暂不考虑学生端自助上传。
- 本项目默认优先支持中文场景，英文教材或双语场景可后续扩展。
- 本项目默认优先满足演示与比赛交付要求，权限、协同和企业化治理能力可后续完善。
- 向量检索能力纳入本期主链路实现，系统接口、数据结构与任务编排需按已接入 Milvus 进行设计。
