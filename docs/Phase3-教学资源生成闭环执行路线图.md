# Phase3 教学资源生成闭环执行路线图

## Phase3 总目标

Phase3 的目标是把 Phase2 已经验证成功的结构化知识和学情基线，继续推进到课堂可用的教学资源成果，形成一条可以演示、可以诊断、可以逐段验收的生成闭环。

本阶段以前端接入和真实链路验证为主，不主动修改后端，不使用 mock 成果兜底。所有成果展示、任务状态、失败原因和下载入口都以真实接口返回为准。

Phase3 不重复《前端详细设计说明书》的完整页面设计，只作为执行路线图使用，帮助后续按真实链路分段推进。

## Phase3-A 自动主链路

### 目标

跑通后端默认自动生成链路：创建生成批次后，自动生成课程大纲、教案和覆盖率报告，并在批次详情页展示真实结果或真实失败原因。

### 实施内容

- 在项目 `#1` 中使用知识版本 `#2` 和学情版本 `#1` 创建生成批次。
- 在批次详情页持续轮询批次详情和关联任务，展示课程大纲、教案、覆盖率生成进度。
- 课程方案 Tab 接入课程大纲列表和详情，展示标题、摘要、课次安排、教学目标、重点难点、学情适配和原始 JSON 兜底查看。
- 教案 Tab 接入教案列表和详情，支持切换课次，展示摘要、教学流程、知识点引用、课后安排和原始 JSON 兜底查看。
- 覆盖报告 Tab 接入覆盖率报告列表和详情，展示覆盖率、告警数、覆盖摘要、报告明细和原始 JSON 兜底查看。
- 若任务失败，优先展示任务状态、错误码、错误信息，并提供 `/tasks/:taskId` 详情入口。

### 验收标准

- 可以从项目工作台成功创建生成批次，创建后进入批次详情页。
- 批次详情页能展示自动主链路的任务状态和进度。
- 至少能看到课程大纲、教案、覆盖率中的真实结果，或清楚看到真实失败原因和任务详情入口。
- `cd frontend && npm run build` 通过。
- 浏览器端打开项目 `#1` 和最新生成批次详情页完成验证。

## Phase3-B 测评链路

### 目标

在课程大纲生成成功后，支持按需创建测评任务，展示测评蓝图、试卷和题目，并提供 DOCX 导出入口。

### 实施内容

- 测评 Tab 根据当前课程大纲展示“创建测评任务”按钮。
- 创建测评任务时使用默认策略：

```json
{
  "scenario_type": "unit_test",
  "scene_type": "unit_test",
  "question_count": 10,
  "question_types": ["single_choice", "fill_blank", "short_answer"],
  "difficulty_range": [1, 5]
}
```

- 接入测评蓝图列表和详情，展示场景、版本、状态、策略摘要、知识点权重、题型分布和难度分布。
- 接入试卷结果列表和详情，展示标题、题量、状态、题目明细、答案解析、来源摘要和原始 JSON 兜底查看。
- 若试卷已有 `export_file_id`，通过文件下载接口提供 DOCX 下载入口。
- 若测评任务失败，展示真实任务错误和任务详情入口。

### 验收标准

- 测评任务可以从批次详情页创建。
- 测评蓝图、试卷和题目能结构化展示。
- DOCX 导出入口在后端返回文件时可用。
- 任务失败时不吞错，不 mock 成果，页面能指向真实任务详情。
- `cd frontend && npm run build` 通过。

## Phase3-C 课件链路

### 目标

在教案生成成功后，支持基于选中教案创建课件任务，展示课件结果，并提供刷新、回复和 PPTX 下载入口。

### 实施内容

- 课件 Tab 根据教案列表选择目标教案，展示“生成课件”按钮。
- 接入课件任务创建接口，创建后刷新批次任务和课件结果列表。
- 接入课件结果列表和详情，展示对应教案、结果状态、页数、模板、页面类型统计、结构摘要和原始 JSON 兜底查看。
- 接入课件状态刷新能力，用于继续查询远程课件生成状态。
- 若远程课件服务需要补充回答，展示回答输入框并调用回复接口。
- 若课件结果已有 `export_file_id`，通过文件下载接口提供 PPTX 下载入口。
- 若课件任务失败，展示真实任务错误和任务详情入口。

### 验收标准

- 可以基于真实教案创建课件任务。
- 课件结果状态、结构摘要、刷新/回复能力可以在页面操作。
- PPTX 下载入口在后端返回文件时可用。
- 外部课件服务失败时，页面展示真实错误，不做假成功。
- `cd frontend && npm run build` 通过。

- Phase3-B：测评任务可创建，蓝图、试卷、题目和 DOCX 导出入口可查看。

- 不提交 `.env`、真实 token、赛题源文件、`node_modules`、`frontend/dist`。

## 风险与后端协作点

- 后端生成批次默认自动触发 `curriculum + lesson_plan + coverage`；若其中任一任务失败，前端先记录真实任务错误，再交给后端确认。
- 测评和课件是按需任务，前端负责触发和展示，但外部 LLM、Embedding、Raccoon、文件导出失败需要后端配合定位。
- 若返回的成果 JSON 结构和设计文档不一致，前端优先展示稳定字段，并使用 JSON inspector 兜底，不在前端臆造结构。
- 若下载接口返回签名地址为空或文件不存在，前端展示真实错误或不可用状态，不提供无效下载按钮。
- 若后端 worker、队列或外部服务不稳定，前端通过任务详情页保留诊断入口，不做重试策略的业务替代。
