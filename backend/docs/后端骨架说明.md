# 后端骨架说明

当前阶段已完成以下基础能力：

- FastAPI 应用启动入口
- 教师登录与当前用户接口
- `/health` 与 `/ready` 健康检查
- MySQL、Redis、Milvus 基础接入
- OBS 存储适配器
- OBS 临时签名下载地址生成能力
- Celery 队列骨架
- MinerU 官方批量解析接口适配层
- 27 表 SQL 对应的 SQLAlchemy metadata 骨架
- Alembic 完整 P0 schema 迁移链
- 已有本地库的 Alembic reconcile / stamp 对齐脚本
- 本地演示教师账号 bootstrap 脚本
- Milvus P0 必需 collection 基线收口
- 教材全量解析、页级重解析、解析人工修正版本
- 学情真实抽取、学情版本列表、学情人工修正版本
- 解析版本显式确认接口
- 知识抽取任务、知识版本列表与详情
- 章节树、知识点列表与知识点详情
- 知识人工修正版本与知识点归并
- OpenAI 兼容 LLM 与独立 Embedding 适配层
- 通用文件下载地址接口

当前基础收口约束如下：

- 业务接口继续保持统一响应壳、全局异常处理和 JWT 依赖注入风格
- 新环境统一通过 `alembic upgrade head` 初始化数据库
- 已由 SQL 脚本初始化的旧库统一通过 `python scripts/reconcile_alembic.py` 接入 Alembic 版本链
- 本地联调统一通过 `python scripts/bootstrap_local.py` 初始化演示教师账号
- Milvus 只自动初始化 `textbook_chunk_vector`、`knowledge_point_vector` 两类 P0 必需集合
- 阶段一默认通过 Worker 轮询 MinerU 结果，不使用回调模式
- 解析与学情人工修正采用版本级提交，不做块级 patch
- 知识抽取只允许从 `parse_status=success` 且 `review_status=confirmed` 的解析版本发起
- 知识人工修正采用操作补丁方式生成新版本，不直接覆盖历史知识版本

当前已进入第二阶段知识结构化实现，后续建议优先继续推进 `curriculum / lesson_plan / assessment / courseware / pipeline / coverage / audit`。
