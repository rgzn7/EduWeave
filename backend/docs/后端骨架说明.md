# 后端骨架说明

当前阶段已完成以下基础能力：

- FastAPI 应用启动入口
- 教师登录与当前用户接口
- `/health` 与 `/ready` 健康检查
- MySQL、Redis、Milvus 基础接入
- OBS 存储适配器
- Celery 队列骨架
- 27 表 SQL 对应的 SQLAlchemy metadata 骨架
- Alembic 完整 P0 schema 迁移链
- 已有本地库的 Alembic reconcile / stamp 对齐脚本
- 本地演示教师账号 bootstrap 脚本
- Milvus P0 必需 collection 基线收口

当前基础收口约束如下：

- 业务接口继续保持统一响应壳、全局异常处理和 JWT 依赖注入风格
- 新环境统一通过 `alembic upgrade head` 初始化数据库
- 已由 SQL 脚本初始化的旧库统一通过 `python scripts/reconcile_alembic.py` 接入 Alembic 版本链
- 本地联调统一通过 `python scripts/bootstrap_local.py` 初始化演示教师账号
- Milvus 只自动初始化 `textbook_chunk_vector`、`knowledge_point_vector` 两类 P0 必需集合

建议后续第二阶段直接在 `backend/app/modules/` 下按业务模块继续扩展，优先实现 `project / textbook / learner_profile / task_center / parsing`。
